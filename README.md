
# RoastLang  ‚Äî Mini Language using Flex & Bison (Lex & Yacc)

RoastLang is a custom mini programming language built using **Flex (Lex)** and **Bison (Yacc)** in **C**.  
This project implements the core phases of a compiler front-end:

1. **Lexical Analysis (Tokenization)** using Flex  
2. **Syntax Analysis (Parsing)** using Bison  

It was developed as part of a **Compiler Design** mini project.

---

##  Key Highlights
- Lexer (scanner) built using **Flex**
- Parser built using **Bison**
- Works with a **Makefile** for easy compilation
- Modular structure using header files (like `ast.h`)
- Generates executable `roastlang` / `roastlang.exe`

---

##  Tech Stack
- **C**
- **Flex (Lex)**
- **Bison (Yacc)**
- **GCC**
- **Makefile**
- Runs on Windows (Git Bash / MinGW) and Linux

---

##  Files in this Repository
| File | Description |
|------|-------------|
| `ast.h` | Header file for helper structures (AST/definitions) |
| `lex.yy.c` | Lexer source file generated by Flex |
| `roastlang.tab.c` | Parser source file generated by Bison |
| `roastlang.tab.h` | Parser header file generated by Bison |
| `Makefile` | Builds and links the project automatically |
| `roastlang.exe` | Generated executable (Windows) |
| `E220_E225 Project Report.pdf` | Project documentation/report |
| `*.o` | Object files generated after compilation (should be ignored in GitHub) |
